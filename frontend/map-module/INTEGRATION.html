🏥 MEDIVISION - Hospital Finder Map

A Complete Integration Guide

📋 Overview

The MEDIVISION Hospital Finder Map is a modern, responsive web-based map module that helps users discover nearby hospitals, clinics, and pharmacies in Pune, India. It's built with vanilla JavaScript, Leaflet.js, and OpenStreetMap — no API keys required!

✨ Key Highlight: This module is completely free, open-source, and requires NO external API keys or paid services. All data comes from OpenStreetMap and related free APIs.

🚀 Integration Methods

Quick & Easy

Method 1: Standalone HTML

Open the map in a separate HTML file or modal

<!-- Open in new tab/window -->
<a href="/map-module/index.html" target="_blank">
  View Map
</a>

Pros: Simplest implementation

Cons: Separate window/tab

Recommended

Method 2: React Component (Iframe)

Embed map in React component using iframe

<iframe 
  src="/map-module/index.html" 
  style={{
    width: '100%',
    height: '600px',
    border: 'none',
    borderRadius: '10px'
  }}
/>

Pros: No state conflicts, isolated

Cons: Limited communication between React and map

Advanced

Method 3: Script Injection

Load map.js directly in React component

useEffect(() => {
  const script = document.createElement('script');
  script.src = '/map-module/map.js';
  document.body.appendChild(script);
}, []);

Pros: Full control and customization

Cons: Requires careful state management

Next-Level

Method 4: Custom React Component

Port map.js logic to React hooks

function HospitalMap() {
  useEffect(() => {
    initializeMap();
  }, []);
  return <div id="map" />;
}

Pros: Full React integration

Cons: Most complex implementation

⚡ Quick Start

Step 1: Copy Files

# Copy the entire map-module folder to your frontend/public
frontend/public/map-module/
  ├── index.html
  ├── style.css
  ├── map.js
  └── README.md

Step 2: Start Local Server

# Using Python 3
python -m http.server 8000

# Or using Node.js
npx http-server

Step 3: Open in Browser

http://localhost:8000/map-module/

Step 4: Integrate with React

Add to your hospitals page component:

// frontend/src/pages/Hospitals.jsx
import React from 'react';

export default function Hospitals() {
  return (
    <div style={{ height: '100vh' }}>
      <iframe 
        src="/map-module/index.html"
        style={{
          width: '100%',
          height: '100%',
          border: 'none'
        }}
        title="Hospital Finder Map"
      />
    </div>
  );
}

🎨 Customization Examples

Example 1: Change Default Location

Edit map.js in the MapManager constructor:

// Change from Pune to Mumbai
this.PUNE_LAT = 19.0760;       // Mumbai latitude
this.PUNE_LNG = 72.8777;       // Mumbai longitude
this.ZOOM_LEVEL = 12;

⚠️ Important Notes

⚠️ Geolocation HTTPS: For production deployment, ensure your app runs on HTTPS. Geolocation API requires secure context (HTTPS or localhost).
ℹ️ CORS Policy: All external APIs used have CORS headers enabled for public access. No backend proxy needed.
ℹ️ Data Freshness: Hospital data comes from OpenStreetMap, which is community-maintained. Data freshness varies by region. Encourage users to report updates to OSM.

🔄 Performance Optimization Tips

  1. Caching: Store fetched hospital data in localStorage to reduce API calls
  2. Lazy Loading: Load more hospitals only when zoomed in or on demand
  3. Clustering: Already enabled - groups 10+ nearby markers automatically
  4. Debouncing: Search input is debounced (300ms) to reduce API calls
  5. Responsive Images: Use emojis instead of image files for markers

🐛 Troubleshooting

Map Not Loading?

  • ✓ Ensure you're opening from a server (not file://)
  • ✓ Check browser console (F12) for errors
  • ✓ Verify Leaflet CDN is loading (Network tab)

Geolocation Not Working?

  • ✓ Allow browser permission when prompted
  • ✓ Use HTTPS or localhost
  • ✓ Check browser location settings

No Hospitals Showing?

  • ✓ Check filters - ensure at least one is checked
  • ✓ Verify OSM data exists for your region
  • ✓ Try zooming out for broader results
